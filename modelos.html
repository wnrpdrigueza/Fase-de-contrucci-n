Identificar el concepto de modelos de software y de la misma forma destacar los más utilizados en la vida actual

El uso de modelos es muy relevante porque ayuda al ingeniero de software a visualizar el sistema a construir, así como también los modelos de un nivel de abstracción mayor pueden utilizarse para la comunicación con el cliente, además pueden ayudar a verificar la corrección en el modelo.


Algunos modelos de software mas destacados son
El modelo en cascada
El modelo de desarrollo evolutivo (espiral)
El modelo de desarrollo basado en componentes
En este modelo se destacan las actividades más relevantes y fundamentales del proceso de especificación, desarrollo, validación y evolución. Los representa como fases separadas del proceso, tales como la especificación de requerimientos, el diseño del software, la implementación, las pruebas, entre otras.
Este modelo se basa principalmente porque enlaza las actividades de especificación, desarrollo y validación. Es decir, surge de un sistema inicial que se desarrolla rápidamente a partir de especificaciones indefinidas.
El proceso de desarrollo se enfoca en integrar estos componentes en el sistema más que en desarrollarlos desde cero. Este desarrollo permite al ingeniero reutilizar piezas de código preelaborado que permiten realizar diversas tareas, conllevando a diversos beneficios como las mejoras a la calidad, la reducción del ciclo de desarrollo y el mayor retorno sobre la inversión.
